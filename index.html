Workplan

11.05., by tinnyx

The main purpose of this project is to visualize Camel route configuration.

First step was to create a parser that is able to parse route from Camel file. Most basic version of it is already finished and more advanced one is in works.

Next step is generating SVG images that visualize routes parsed from Camel file. When finished, we should be able to generate three different boards (RasPi B+, BeagleBoneBlack and CubieBoard 2) and also every needed component.

After that, next step is creating graphical interface so this funcionality can be used effectively. When we have first prototype of application working, we will work on testing and possibly adding more features.

Currently, not all roles are fully fleshed out and assigned, but Norbert Slivka is working on aforementioned parser and Martin Zilak is beggining his work on the SVG generator.
